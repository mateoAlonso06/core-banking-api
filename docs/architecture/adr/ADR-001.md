# ADR-001: Representación de dinero mediante Value Object Money

## Status
Accepted

## Context
El dominio bancario maneja montos monetarios que requieren:
- Precisión decimal
- Validación de moneda
- Reglas de redondeo consistentes
- Invariantes de no negatividad

El uso directo de BigDecimal dispersa reglas de negocio y permite inconsistencias.

## Decision
Se decidió introducir un Value Object `Money` que encapsula:
- BigDecimal
- MoneyCurrency
- Reglas de escala y redondeo
- Validaciones de moneda

## Consequences
### Positive
- Reglas centralizadas
- Mayor expresividad del dominio
- Reducción de bugs financieros
- Facilita testing

### Negative
- Mayor complejidad inicial
- Necesidad de mappers para persistencia
